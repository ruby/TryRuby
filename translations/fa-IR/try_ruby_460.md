می‌بینی، آکولادهای خالی {} در واقع کوتاه‌شده و میان‌بری برای Hash.new هستن، متد new برای ساختن <ruby>شی<rt>object</rt></ruby> از یه کلاس مشخص استفاده می‌شه؛ یه جورایی کلاس یعنی یه گروه مشخص از شی‌ها که شبیه هم هستن، کارهای مشابه انجام می‌دن، لباس‌های مشابه دارن و…
فایدهٔ یه کلاس چیه؟

### Blurbalizer<sup>TM</sup>
تصور کن یه ایده کاربردی برای یه برنامه جدید داری، این برنامه قراره سکوی بعدی پیام‌رسان فوری باشه.
می‌خوای یه برنامه بسازی که مردم بتونن پیام‌های کوتاه بفرستن، این پیام‌ها رو می‌ذاری Blurb<sup>TM</sup> صدا بزنی.
هر Blurb<sup>TM</sup> حداکثر ۴۰ نویسه طول داره. حتی می‌خوای یه تنظیم حالت خلق و خو هم اضافه کنی.

### از کجا شروع کنیم
خب، می‌خوای پیام‌های Blurb<sup>TM</sup> رو تو یه پرونده json ذخیره کنی، درست؟
اما چطور می‌خوای محتوا و زمان ارسال هر پیام رو پیگیری کنی؟
وقتی پرونده رو بارگذاری کردی، توی روبی چطور نمایش داده می‌شه؟
یه Hash خواهد بود؟ یه Array؟ یا یه Array از Arrayها؟ یا یه چیز دیگه؟

### Class
به نظرم بهترین گزینه اینه که یه کلاس بسازی؛ تو قبلا با چند تا کلاس آشنا هستی: 
Hash، Array، String.

    class Blurb
    end
